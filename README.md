# Otgruzka Tovara / Fibercell Manufacturing MVP

MVP для управления производственной цепочкой по пропиточной линии:

1. Прием сырья (сетка, стеклоткань, ратл)
2. Пропитка
3. Контроль качества
4. Склад
5. Отгрузка

## Что в MVP сейчас

- Дашборд с ключевыми показателями (рулоны, метры, партии)
- Ролевой вход через Firebase Auth (email/password)
- Роли пользователей из Firestore (`users/{uid}`)
- Интерфейсы по ролям (каждый видит только свой блок)
- HR блок для роли `hr` и `admin`
- Правила Firestore (`firestore.rules`)

## Быстрый старт

```bash
npm install
npm run dev
```

## Firebase переменные

Создай `.env` из `.env.example`:

```bash
cp .env.example .env
```

Все значения уже внесены в `.env.example`, можно скопировать как есть.

## Учетные записи (Auth)

Боевые пароли и списки пользователей не хранятся в репозитории.
Создание пользователя выполняется через админ-интерфейс, а роль и permissions сохраняются в `users/{uid}`.

## Безопасность

- Пароли не хранятся в коде приложения.
- Доступ к данным ограничивается ролями в `firestore.rules`.
- Для прода рекомендована смена всех стартовых паролей после первого входа.

## One-time шаги в Firebase Console

Перед первым входом нужно включить сервисы проекта:

1. Включить Authentication и провайдер `Email/Password`.
2. Включить Firestore API и создать базу данных `(default)`.
3. После включения применить правила:

```bash
firebase deploy --only firestore:rules --project otgruzka-tovara
```

## Производственная логика (следующий этап)

- Справочник сырья и поставщиков (`SIMO`, др.)
- Карта партии: закупка -> линия -> QC -> склад -> отгрузка
- Нормы расхода пропитки, потери, выход годного
- Акты списания/брака
- Отгрузочные документы и себестоимость
- Ролевой доступ: склад, технолог, ОТК, бухгалтерия, админ
