# Технолог · контроль качества

Документация по переносу **логики и калькуляции** из Excel-файлов в модуль «Технолог» (FiberCell / FST).  
**Исторические данные из Excel не импортируются** — только формулы, поля и журнал новых записей.

## Где в приложении

**Технолог** → вкладка **«Контроль качества»** → подвкладки:

| Подвкладка | Источник Excel | Назначение |
|------------|----------------|------------|
| EAD расчёт | `EAD Расчет.xlsx` | Ячейка суровья, H1 по зонам |
| EAD контроль | `EAD Контроль.xlsx` | Навеска г/м² на линии |
| Входной контроль | `Входной контроль.xlsx` | Приёмка химии и суровья |
| Контроль пропитки | `Пропитка.xlsx` | NV, вязкость после замеса |

## 1. EAD расчёт (`EAD Расчет.xlsx`)

### Поля (как в шаблоне «шаблон»)

- **m0** — масса чашки, г  
- **m1** — масса чашки с пропитанным материалом, г  
- **m2** — масса охлажденной чашки с сухим остатком, г  
- Зоны: **край (л)**, **середина**, **край (пр)** — в каждой m0, m1, m2  

### Формулы (реализованы в `calc.ts`)

**Размер ячейки, мм** (3 замера через запятую):

- Режим **прибор**: `AVERAGE(z₁,z₂,z₃) / 100 / 3 / 20`
- Режим **вручную**: `AVERAGE(z₁,z₂,z₃) / 20`

**H1 по зоне** (органическое содержание / доля по Excel):

```
H1 = (m1 − m2) / (m1 − m0)
```

**Среднее по зонам** (колонка «Органическое содержание»):

```
H̄ = AVERAGE(H1_л, H1_середина, H1_пр)
```

**Остаточная влажность** (как на листах с колонкой «Остаточная влажность»):

```
1 − H̄
```

## 2. EAD контроль г/м² (`EAD Контроль.xlsx`, лист «гм²»)

### Поля

- Продукт, суровьё, линия, дата, **план г/м²**
- Три замера **«л»** и три **«пр»** (сырые значения из Excel)

### Формулы

```
Л.сред = AVERAGE(л₁, л₂, л₃) / 10
П.сред = AVERAGE(пр₁, пр₂, пр₃) / 10
Средняя = AVERAGE(Л.сред, П.сред)
Δ г/м² = Средняя − План
Δ % = Δ / План × 100
```

**Статус**: «Соответствует», если |Δ %| ≤ 5 % (как типичный допуск в таблице); иначе «Не соответствует».

## 3. Входной контроль (`Входной контроль.xlsx`)

### Типы материала

1. **Химия** — pH, сухой остаток %, сравнение с паспортом  
2. **Суровьё** — г/м², ячейка (основа×уток), прочность О/У в Н  
3. **Прочее** — свободный текст результата  

### Логика

- **pH в норме**: `phMin ≤ pH ≤ phMax` → иначе статус «Не соответствует»
- **Отклонение сух. остатка от паспорта** (как формула `(51−44.8)/51` в Excel):

```
Откл. % = (Паспорт − Факт) / Паспорт × 100
```

- Для суровья статус «Соответствует» при заполнении измерений (детальные нормы задаются вручную в полях pH min/max и паспорте).

## 4. Контроль пропитки (`Пропитка.xlsx`, лист «сетка»)

### Поля

- Рецептура (из справочника технолога), № партии, даты, операторы, контролёр  
- **m0, m1, m2** — gravimetric  
- Вязкость: время (с), температура (°C)  
- NV теор. — подставляется из рецепта или вручную  

### Формулы

**NV (сухой остаток), %**:

```
NV = (m2 − m0) / m1 × 100
```

**NV теор. из рецепта** (сухая база / объём партии):

```
NV_теор = dryBatchKg / totalBatchKg × 100
```

**Отклонение** (как в Excel: `ABS(P−J)`, `Q/P`):

```
|Δ| п.п. = |NV − NV_теор|
Отн. откл. = |Δ| / NV_теор
```

**Допуск по умолчанию**: ±5 процентных пунктов (строка «Допуск 5 процентных единиц» в Excel).

**Статус**: «Соответствует», если `|Δ| ≤ допуск`.

## Связь с существующим модулем

| Уже было | Новое |
|----------|--------|
| Рецептуры, замес, NV косвенно через dry/total | Явный контроль NV после замеса |
| `extractSolidsPct(note)` из текста | Структурный входной контроль химии |
| Нет EAD | Полный расчёт H1 и г/м² |
| Журнал замесов | Отдельные журналы QC-записей |

---

## Что сознательно не переносилось

- Листы-периоды с историей (имена дат в Excel) — только шаблон + журнал в приложении  
- Макросы, цветовое оформление, объединённые ячейки  
- Автоподстановка поставщиков и контейнеров из закупок (можно добавить позже через связь с `procurement`)
